# AiBar

Легкое PWA-приложение для подбора коктейлей по настроению и тегам.

## Что умеет

- ведет диалог как бармен;
- задает максимум 3 уточняющих вопроса;
- после этого выдает 3 коктейля;
- для алкогольного режима подбирает:
  - 1 крепкий из базы;
  - 1 менее крепкий из базы;
  - 1 авторский, придуманный моделью или локальным fallback;
- для безалкогольного режима подбирает:
  - 2 из базы;
  - 1 авторский, придуманный моделью или локальным fallback;
- работает как PWA и кэширует основные файлы.

## Запуск

Сайт теперь запускается вместе с backend API, который может ходить в базовую `Qwen` через Hugging Face Router.

Примеры:

```powershell
npm run dev
```

По умолчанию сервер поднимается на `http://127.0.0.1:4173`.

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни:

- `HF_TOKEN`
- `HF_CHAT_MODEL`

Пример:

```powershell
HF_TOKEN=hf_xxx
HF_CHAT_MODEL=Qwen/Qwen3-4B-Instruct-2507
```

Если `HF_TOKEN` не задан, сайт не ломается: авторский коктейль будет собираться через локальный fallback.

## Текущая схема

- чат и базовые рекомендации работают в приложении;
- авторский коктейль сейчас можно генерировать через backend и базовую `Qwen`;
- позже backend можно переключить на дообученную модель без переделки фронта.
